What is the difference between map, filter, forEach and find method?

  • map(): map creates a new array based on the result of applying a function to each element.
  • filter(): filter returns a new array with elements that pass a test.
  • forEach(): forEach performs a function on each element.
  • find(): find returns the first element in an array that passes a test.

Difference among local storage, session storage and cookies?

  • Cookies: Cookies are small text files that are stored on the client-side by a web browser. They are designed to store small amounts of data for a long period of time and can be accessed by both the client and server. They are commonly used to store user-specific information and can be accessed across multiple sessions.
  • Local Storage: Local storage is a mechanism that allows web applications to store larger amounts of data on the client-side. Data stored in local storage remains even after the browser is closed and can only be accessed by the client. It can be used to store data such as user preferences, cached data, or application state.
  • Session Storage: Session storage is similar to local storage, but the data stored in session storage is cleared when the session ends. A session ends when the user closes the browser or after a certain period of inactivity. Session storage can be used to store data that should only be available for a single session, such as shopping cart data or temporary user inputs.

What is event loop and how does it works?

The event loop is a mechanism in JavaScript that allows the browser to handle multiple tasks simultaneously without blocking the main thread. It works by continuously checking for new tasks to execute and processing them in a non-blocking way. The tasks are stored in a queue, and when a new task is added, it is moved to the call stack for execution. The call stack is a stack of function calls, and when a function is executed, it is added to the top of the stack. The event loop continues executing functions on the call stack until it is empty, and then moves the next task from the queue to the stack, and the process repeats.
